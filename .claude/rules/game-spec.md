# ゲーム仕様（ポケモンチャンピオンズ）

ドメイン計算とデータが従う**確定ゲーム仕様**の要点。正本は `docs/plan/01-mvp/architecture.md`（「確定しているゲーム仕様」節）。ここは計算式の取り違えを防ぐための要約で、詳細・出典はそちらを参照する。

## 固定前提

- **レベル 50 固定・個体値 31 固定**（個体ごとの調整余地は能力ポイントと性格のみ）。
- **能力ポイント制（努力値廃止）**: 合計 **66**、1 能力あたり最大 **32**。
- **性格補正** = ×0.9 / 1.0 / 1.1。上げ能力・下げ能力を自由選択でき、**同一能力に両補正は不可**（up = down は不正）。
- **現行レギュレーション = M 系**: メガシンカ**可**、テラスタル / ダイマックス**不可**。

## 実数値の計算式（`src/domain/calc-stats.ts`）

Lv50・個体値31 を代入済みの形:

- **HP** = `floor((種族値×2 + 31 + 能力ポイント×2) × 50/100 + 60)`
- **HP 以外** = `floor( floor((種族値×2 + 31 + 能力ポイント×2) × 50/100 + 5) × 性格補正 )`

**HP 以外は二重 floor**（内側の floor 後に性格補正を掛けてさらに floor）。順序を変えると 1 ずれる。テストは二重 floor の端数が出る境界と、ポイント **0 / 32** の境界を重点的に網羅する（カバレッジ 100%）。

## 耐久指数 / 火力指数（`src/domain/stat-indices.ts`）

ステータス調整の壁打ちで物理 / 特殊の寄りや火力の伸びしろを比較するための無次元の近似指標。

- **耐久指数** = HP実数値 × 防御系実数値（物理 = HP×防御 / 特殊 = HP×特防）。被ダメージは防御に反比例し、確定数は HP に比例するため、HP×防御 が確定数の伸びの近似になる。
- **火力指数** = 攻撃系実数値 × 技威力（物理 = 攻撃×威力 / 特殊 = 特攻×威力）。素ダメージ（乱数・タイプ補正前）に比例する近似。

## ダメージ式（`src/domain/damage.ts`・確定数判定の範囲に限定）

MVP では**確定耐え / 確定数の判定に必要な範囲**だけ実装する（急所 / 天候 / 持ち物 / 特性補正は将来計画 `02-<slug>`）。Lv 既定 50:

- 素ダメージ = `floor( floor( (floor(2L/5)+2) × 威力 × 攻撃 / 防御 ) / 50 ) + 2`（防御で割った後に floor、その後 50 で割って floor の順序を守る）。
- → STAB（×1.5 floor）→ タイプ相性（×倍率 floor）→ 乱数（×0.85〜1.00 floor）。
- **乱数最大（×1.00）= `maxDamage`**（確定耐え判定 = HP > maxDamage）、**乱数最小（×0.85）= `minDamage`**（確定数判定）。

## ポイント逆算（`src/domain/stat-tuning.ts`）

目標ライン（「素早さ □□ 抜き」「攻撃 ○○ の技を確定耐え」）から能力ポイント配分を逆算する。実数値は能力ポイントに対し単調増加なので、0..32 を昇順に走査して目標到達の**最小振り**を求める（`minPointForTarget`）。

- **素早さ □□ 抜き**: 目標素早さ実数値以上に届く最小素早さ振り（相手を抜くなら `相手の素早さ + 1` を渡す）。
- **確定耐え**: 各 HP 振りに対し耐える最小防御振りを求め、合計振り昇順の**フロンティア**を候補列挙する（解は合計66の中で一意でないため）。
- **実現不能**（32 振りでも届かない / 耐えられない）は `null` で報告する。
- 逆算結果は `calc-stats`（`calcHp` / `calcStat`）に通すと目標ラインを満たすことをテストで**往復検証**する。

## 留意

- HP には性格補正が掛からない（補正は HP 以外の 5 能力のみ）。
- メガ個体は素種族で定義し、`megaEvolvesTo` 先の種族値 / タイプで追加分析する（[[type-conventions]] 参照）。
- 耐久 / 火力指数・ダメージ式・逆算は **近似 / 限定スコープ**。本格ダメージ計算（乱数16段の確定数表・急所・補正）は将来計画 `02-<slug>` へ。
- 仕様変更時は ADR + `architecture.md` + 本 rule をセットで更新し、乖離させない。
