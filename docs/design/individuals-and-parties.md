# 個体・パーティ管理の仕組み

pokeform は育成済み個体とパーティを「**コードとして**」扱い、その正しさを型検査で保証する。この doc は
個体・パーティが**保証する不変条件**と、その上に乗る分析の設計意図を説明する（具体値・数式・フィールド名・
CLI フラグは持たない。正本は末尾「実装 SoT ポインタ」へ）。

## 個体・パーティを「コードとして」検証する

利用者は個体を YAML、パーティを Markdown で書く。これらは型バリデーションの仕組み（[type-validation](./type-validation.md)）に
従い、生成コードへ変換されてから型チェックされる。「正しい構築だけが型を通る」状態を作ることで、
構築のブラッシュアップを**型エラーの解消として機械的に回せる**。実数値などの導出値は型を通った個体に対して
純粋なドメイン関数で計算し、入力の不備（誤った実数値の暗示表示など）を黙認しない。

## 保証する不変条件

個体・パーティが満たすべき制約を**自然言語で列挙**する。各制約は違反すると意味の分かるエラー型として
診断に現れる（[type-validation](./type-validation.md)）。**閾値・計算式の実体は持たない**——具体値は [[game-spec]]、型表現は
[[type-conventions]] が正本。

**個体（1 体）について保証すること**:

- **覚えない技を持てない** — その種族が（対象レギュレーションで）習得しない技は弾く。
- **使えない特性を持てない** — その種族が持たない特性は弾く。
- **持てない持ち物を持てない** — レギュレーションの解禁プールにない持ち物、メガ形態に対応しないメガ
  ストーンなどは弾く。
- **能力ポイントの配分が規定を満たす** — 合計が規定値ちょうどであること、各能力が上限を超えないこと。
  具体的な合計値・上限値は [[game-spec]] を正本とする。
- **性格補正が整合する** — 上げ能力と下げ能力が同一でないこと（同一能力への両補正は不正）。

**パーティ（最大 6 体）について保証すること**:

- **同種族を重複させない** — 同じ種族を複数編成しない。
- **レギュレーションに解禁された種族だけ** — 宣言したレギュレーションの解禁集合に含まれる種族のみ。
- **体数が上限以内** — 規定体数を超えない。
- **参照が解決する** — メンバーとして参照する個体ファイルが実在し、読み込める。
- **メンバーが当該レギュレーションを宣言している** — パーティの宣言レギュレーションを、各メンバー個体も
  対象として宣言していること（型検証されていない不整合の防止）。

解禁は**レギュレーション側を一本の正本**にして判定する。種族側に解禁フラグを散らさないことで、レギュレーション
追加・更新時に直す場所を一意にする（[ADR 0021](../adr/0021-per-regulation-species-and-legality.md)）。

## 入力言語はファイル単位で宣言する

個体 YAML・パーティ Markdown は、**ファイル単位でその記述言語（日本語 / 英語）を宣言**する。loader は
宣言言語に従って全名称フィールドを正規の ID へ正規化してから検査する。宣言言語と食い違う名称や未知の
名称は、行番号付きで報告する。**表示言語は入力言語とは独立**に指定でき、入力が何語でも診断・分析表
の表示言語を別途選べる。狙いは、言語混在の曖昧さを排除してエージェントにとって予測可能にすること。
背景は [ADR 0014](../adr/0014-yaml-lang-per-file.md)、具体仕様は [[cli-and-io]]。

## CLI でできること（概念）

CLI はファイル単体・ディレクトリ（再帰）のどちらも受け取り、**問題を検出したら非 0 で終了**する
（CI / エージェントが合否を機械判定できる）。概念的には次のことができる。

- **個体の整合チェック** — 上記「個体について保証すること」を型検査で確かめる。
- **パーティの整合チェック** — 上記「パーティについて保証すること」を確かめる。
- **パーティの分析** — 技範囲と防御弱点を集計する（次節）。
- **ステータスの壁打ち** — 実数値・性格補正・耐久 / 火力の近似指標を表示し、目標ラインからポイント配分を
  逆算する補助。

コマンド名・フラグ・終了コードの具体は持たない（[[cli-and-io]] が SoT。手順は `author-individual` /
`review-party` / `stat-tuning` skill）。

## 分析が答える問い — 技範囲と防御弱点

パーティ分析の価値は、構築の穴を**定量的な問い**として可視化することにある。

- **技範囲の穴**: パーティ全体の攻撃技で、等倍超のダメージを出せない防御タイプはどれか。
- **防御弱点の集中**: ある攻撃タイプに対し、何体が弱点を負い、何体が半減できるか。弱点が一定数以上に
  集中すると脆弱として警告する。

メガを持つ個体は、メガ後のタイプでも追加で分析する。相性倍率の算出規則・しきい値・耐久 / 火力指標の
定義といった**数値の実体は持たない**（[[game-spec]] が正本、計算は `src/domain/` の純関数）。

## 設計上の注意（実体は下層）

- **実数値は二重 floor**: HP 以外の実数値は、内側の floor のあとに性格補正を掛けてさらに floor する。
  順序を変えると 1 ずれるため、計算は単一のドメイン関数に集約する。式の実体は [[game-spec]]。
- **導出値は型を通った入力に対してのみ計算**し、表示専用の経路でも入力の不正を黙認しない。
- **耐久 / 火力指標・ダメージ判定は近似 / 限定スコープ**。厳密なダメージ計算は将来計画の領域（[[game-spec]]）。
