## ⚠️ Problem（詰まった点・制約）

- **spread 合成が per-reg dex へ global `megaEvolvesTo` を漏らす silent 値回帰**になりかけた。`{ ...speciesSpecsDex[sid], ... }` は species-specs の global mega 能力を含むため、per-reg で mega 未解禁の M-B garchomp に [garchomp-mega] が混入する。実装中に入れた防御 throw で検知し、base エントリを明示フィールド参照（spread 不使用）へ切替えて回避したが、防御チェックが無ければ tsc 緑のまま値が変わっていた。
- **Biome `useLiteralKeys` が生成コードの computed-key access を error 扱い**にし、generate 初回が lint 違反になった。`speciesSpecsDex["dragapult"]`（識別子）は dot へ矯正要求・`["rotom-wash"]`（ハイフン）は対象外、と混在するため、識別子/非識別子で出し分ける `acc()` ヘルパを generate に足す必要があった。
- **`.githooks/pre-commit` のハードコード旧パス `data/champions/regulations` が commit 時まで露見しなかった**。このゲートは `pnpm verify` に含まれず（commit 境界の高速ゲート）、typecheck/test/lint/yaml-style が全緑でも commit が ENOENT で落ちた。レイアウト移動時にハーネスゲートのパス追従が verify の死角に入る。
- **廃止シンボル `speciesBaseDex` の doc コメント追従漏れ**（`individual.ts` は更新したが `load-individual.ts` / `regulation.ts` の JSDoc が旧名参照のまま）。code-review nit で発見。コード参照は tsc が捕るが、コメントは機械ゲートの死角。
- **`scripts/*` は tsconfig 対象外で typecheck されず verify にも乗らない**。`generate`/`materialize`/`fetch-pokeapi`/`serebii-to-catalog` の orchestrator 改修は型検査も自動テストも掛からず、特に中間 JSON が要る `serebii-to-catalog` の新ツリー書き出しは smoke-run できず import 解決のみで確認した（純関数は src 側で 100% カバレッジ）。
