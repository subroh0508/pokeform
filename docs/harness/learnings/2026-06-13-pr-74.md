## ⚠️ Problem（詰まった点・制約）

<!-- 最低 3 / 推奨 5-10 -->

- **ポインタ式 README は SoT(rule) 変更時に追従漏れが起きうる**: 取得元欄の鮮度が rule 側の方針転換に依存する。設計上は rule 変更が README のリンク先で吸収されるが、PokeAPI 正式対応など方針転換時に README の「取得元 / SoT」欄の文言が古いまま残るリスクは残る。
- **「何を表すか」列のフィールド名列挙がドリフトしうる**: harness-review が nit として記録した通り、`stats`(H/A/B/C/D/S) や技メタ `type/damageClass/power/accuracy/pp/priority` などフィールド名を列挙しており、スキーマ複製ではない（識別ラベル）ものの、将来フィールドが増減した際に rule とドリフトしうる微小リスクがある。
- **方針転換時のチェックポイントが仕組み化されていない**: 「rule を直せば README のリンク先で吸収される」という設計依存で、取得元欄など README 固有の鮮度を保つ明示的なトリガ（例: data-pipeline.md / PokeAPI 方針を変える PR で README を見直す）がチェックリスト化されていない。
