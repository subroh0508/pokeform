## ✅ Keep（継続したいこと）

- **明示データ駆動へ機構を一本化した**: メガ形態の legality を「型レベル暗黙逆引き（`MegaStoneOf<S>`）」から「生成 `items` タプル + `Extract`」へ移し、`HoldableItems` を 3→2 分岐に簡素化。「明示タプル vs 暗黙 MegaStoneOf」の二重機構（取り違えの温床）を解消した。
- **回帰を X/Y 精密区別まで実証した**: メガ形態 `charizard-mega-x` が `charizardite-y`（他形態ストーン）/ `life-orb`（通常持ち物）→ いずれも `ItemNotHoldableBy`、`charizardite-x` のみ通過を probe で確認。base 種族の通過も併せて検証。
- **fail-fast を generate へ寄せた**: reg 未解禁ストーンの排除を型レベル `& RegulationItemId<R>` から `megaFormStones` の例外（reg プール外/欠落）へ移し、データ不整合を生成段で早期検出する形にした。
- **決定論を md5 で機械確認**: 連続再生成で m-a/m-b index.ts の md5 一致を確認し、生成の冪等性を担保。
- **撤去に伴う波及を漏れなく追従した**: `MegaStoneOf` 撤去で未使用化した import（`ItemSpecsDex`）・`item.ts` の `megaSpecies` 用途注記・rule 2 ファイルを grep で洗って同期。

## ⚠️ Problem（詰まった点・制約）

- **同一セマンティクスが 3 連続で揺れた（#145→#147→#149）**: Phase 6 で base 専有 → #147 で base 解除（型側 MegaStoneOf に依存）→ #149 でメガ形態に明示タプル。#147 時点で「データ上に制限が見えない」ことに気づけず、ユーザ指摘で再修正になった。「型で効いている」と「生成データに現れている」の差をレビューで切り分けられていなかった。
- **メガ形態 legality を gate する committed test/fixture が依然不在**: 制約は型レベルで line カバレッジ 100% だが、`check:individual` に乗る fixture/test が無く、3 度の変遷を通じて回帰を機械的に止められていない。検証は毎回手動 probe 頼り。

- **完了済み plan doc が機構変遷に追従していない**: plan 03 phase-07 が削除済み `MegaStoneOf<S>` を、plan 04 が base タプル方式を実装事項として記述したまま。mutable WIP の歴史記録だが現状と乖離。
