- **層1 決定論スクレイパー + gate 駆動の収束ループ**: `scrape:serebii species` → `serebii:catalog` → `check:regulation` が報告する不足 id を `move-master` で補完、を反復して 252 技を欠落 0 へ収束させられた。HTML を読まず exit code / 不足 id だけで駆動できた。

- **メガ自動著述（ADR 0033）が新規 mainline メガで動作**: tyranitar-mega / lucario-mega を per-reg / mega-specs / megaEvolvesTo / megaStone へ決定論で起こせた。

## ⚠️ Problem（詰まった点・制約）

- **Champions 固有メガ/ストーンが materialize 不可**: Serebii は本作新規メガ（Mega Starmie/Mega Dragonite）を掲載し skill は自動著述するが、PokeAPI（vendor・ADR 0027）に当該 slug が無く `fetch:data` が 404 で停止。構造データ（dex/types/stats/ability）を埋める経路が無く、本フェーズでは投入を断念した（mainline メガのみ投入）。
- **新規種族の null 構造 placeholder で materialize crash**: `serebii:catalog species` は新規種族を species-specs に null で登録するが、`materialize` の `planFields` が null 非対応で `Cannot read properties of null`。既存種では未顕在、新規 starmie 投入で初めて露見。空 map で回避したがパイプラインの素の不整合。
- **空 regulation を表現できない**: per-reg を空にすると `species: []`/`mega: {}` が flow 扱いで `check:yaml-style` に弾かれ、空 block コレクションは YAML で表現不能。結果 m-b を「空リセット」できず削除（Phase 2 再構築）へ倒さざるを得なかった。phase doc の「m-b を初期状態へ戻す」と gate が非互換。
- **team サンプルとロスターの暗黙結合**: `.githooks/pre-commit`(`check:individual team/`)・`pre-push`(`typecheck standard.md`) が team を実質ゲートするため、ロスターを絞ると team 参照種/レギュが連動破綻する。phase doc は team 連動に言及せず、roster 選定が team 制約に縛られた（rotom-wash のフォルム slug 404 まで巻き込んだ）。
- **mega ability / mega ja が手動著述依存**: `serebii:catalog` はメガを baseSpecies のみで起こし、`materialize` は ability/ja を埋めない。新規メガで sand-stream/adaptability・メガバンギラス/メガルカリオを手動補完した。
- **削除レグの stale 生成物が残る**: m-b 削除後も `src/generated/champions/m-b/` が残り tsc が落ちた（`generate.ts` は除去レグのディレクトリを掃除しない）。手動 `git rm` が必要だった。
- **roster-source doc を Write した際に tool-call ラッパー（`</content>`）が末尾混入**し harness-review で blocking 検出。長文 doc の Write で起きやすい。
