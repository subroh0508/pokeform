## ✅ Keep（継続したいこと）

- **抽出層（showdown ツリー依存）と転記層（純関数）を明確に分離**した。`scripts/showdown/*`（`../../sim/dex` 依存・tsc/coverage 除外）に副作用と外部依存を閉じ込め、判断分岐を持つ id 正規化・写像は `src/codegen/showdown/*-fields.ts`（カバレッジ 100%）へ抽出。[[testing]] の「除外パスへ意味ある純関数を置かない」を設計時に満たせた。
- **等価検証を「重複 id の値突き合わせ」として実施**した。既存 committed が placeholder subset（10 種）ゆえ単純 git diff では値の不一致が planFields の既存尊重で隠れる。一時スクリプトで showdown 派生値 vs committed を直接照合し、76 species / 740 moves / 148 items / 17 mega / 42 learnset で不一致 3 件（全て説明可能）と定量確認できた。
- **PP の calculatePP 落とし穴を抽出層に閉じ込め**、740 技すべての技メタが既存と完全一致することで経路の正しさを実証した（§3.4 の最重要落とし穴を de-risk）。
- **両ソースの 5 データセット軸（species/moves/items/abilities/mega）対称**を抽出・転記・package.json scripts で一貫させ、後続 Serebii 実装の写し鏡にできる土台を作った。
- **独立レビューが latent defect を捕捉**した。`data/` 未変更で CI 緑・等価検証も通る死角にあった `syncMega` の二重ネストバグを code-review が指摘し、実機（`pnpm showdown:mega m-a`）で是正確認してからマージできた。

## ⚠️ Problem（詰まった点・制約）

- **転記の配線（`scripts/sync-showdown.ts`）が coverage/typecheck 両方の死角**に落ちた。scripts/ は tsc include 外かつ coverage 除外のため、`syncMega` の YAML 二重ネストバグは機械ゲートでは一切顕在化せず、レビューと実機実行でしか捕れなかった。等価検証も JSON 値の照合に留まり YAML write 経路を通らなかった。
- **`applyFill` が planFields の conflict を握りつぶす**（materialize は warn する）。Phase 1 では既存尊重で実害ないが、Phase 6 の全量投入で showdown と committed の値が食い違っても黙って既存を残すため、差分検出が効かない懸念が残る（non-blocking 指摘）。
- **`kebabId` がアクセント文字種で PokeAPI id と乖離**しうる（Flabébé→`flab-b-` 等）。champions roster には現れないが、将来レギュで顕在化する潜在リスク。
- **showdown ツリーの build 出力先（dist/）と import 深度（`../../sim/dex`）の前提が暗黙**で、最初 `../sim/dex` / `tools/showdown/cli.js` 直参照を誤り、実機で `dist/tools/showdown/cli.js` と判明してから修正した。配置規約が doc 化されていないと再現時に詰まる。
