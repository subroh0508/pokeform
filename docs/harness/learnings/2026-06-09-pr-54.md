- **スモークテストで e2e 確認**。degrade（raw 無）/ full（raw 有）/ 不正技 fixture（exit1）の 3 系統で

- **独立レビュー 2 種（code-review / harness-review）で blocking 0**。domain/CLI 分離・ADR 採番・SoT 整合を

## ⚠️ Problem（詰まった点・制約）

- **learnset 検証の CI 非対称**。覚えない技検証は data/raw 依存で pre-commit 中心。CI では走らないため、
  raw を持たない経路での編集には理論上の抜けが残る（generate も raw 必須なので実害は小）。
- **`loadLearnsets` が破損 raw JSON を無検証**（レビュー non-blocking）。vendor 制御下で実害小だが
  「raw あり」と「整形済み」を区別できると堅牢。
- **条件適合検証（基本最終進化・Restricted 除外）が未対応**。進化チェーンデータが要るため本 phase 対象外で、
  参照整合 + 覚えない技に絞った。
- **schema 検証の push 順序**（mega 非配列時の bad-species-block が moves 検証 issue と混在しうる・non-blocking）。
  挙動上の害は無いが一貫性に改善余地。
