## ⚠️ Problem（詰まった点・制約）

- **legality モデルの帰結で既存個体が暗黙に不正化した**: 「メガ可能種は対応ストーンのみ」により garchomp（focus-sash）が不正に。これは `pnpm verify` の非ゲート対象（team は `check:individual`/`check:party` 専用）なので verify では顕在化せず、着手時に grep で影響を洗う必要があった。
- **src fixture が verify 上の唯一の破壊点だった**: `src/types/individual.test.ts` の garchomp item は verify でゲートされるため見落とすと typecheck 赤。team データ（非ゲート）と src fixture（ゲート）の二重影響を切り分ける必要があり、影響範囲の事前棚卸しが効いた。
- **plan doc / OVERVIEW / README が実型名と食い違っていた**: 概念名 `ItemNotHeldBy` に対し実型は `ItemNotHoldableBy`。回帰確認の期待値が doc と一致せず、追跡性ギャップ（実害なし）になった。harness-review も nit として検出。
- **doc コメントが旧モデルのまま残っていた**: `HoldableItems` の base 種族分岐コメントが「base = 全プール」のままで、emit 値変更により不正確化。doc-data 乖離を同 PR で追従する必要があった。
- **rule のエンティティ定義が新追記の依存フィールドを列挙していなかった**: 追記した legality 記述が `item-specs.megaSpecies` に依存するのに、data-pipeline の item-specs フィールド列挙には `megaStoneFor?`/`category` のみで `megaSpecies` が欠落。harness-review nit を受け追補した。
