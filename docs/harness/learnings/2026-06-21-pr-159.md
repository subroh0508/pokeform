## ⚠️ Problem（詰まった点・制約）

- **素の phase 番号の renumber 置換でバグを作り、検証で見逃した**: 06→09 置換時に perl の lookahead `(0[45] → )06(?![→0-9])` が「06 の直後の空白」を許し、`02/OVERVIEW:332` の plan 内部 phase チェーン（`05 → 06 → 07`）を `05 → 09 → 07` に破損。さらに検証 grep で当該行を `grep -v ...:332` で**除外していた**ため初回は見逃し、次ラウンドの 09→XX 追従中に発覚した。「壊しうる行を検証から除外する」のが見逃しの直接原因。
- **採番・実施順・スコープの方針が 2〜3 転して大きく手戻りした**: 初回 06→09(retired) → レビューで 07/08 入替 + XX 採番へ変更。ディレクトリ swap・全 cross-plan 参照の再追従・OVERVIEW/phase の依存方向反転が丸ごと再作業になった。着手前に採番・順序・スコープを確定していれば避けられた手戻り。
- **ローカル調査メモを誤ってリポジトリにコミットした**: `refactor-survey-2026-06-21.md` を「OVERVIEW のリンク解決のため」同梱したが、本来リポジトリ管理外の素材だった。OVERVIEW/phase から `§` 参照リンクを張ったため self-contained でなく、削除時に全 `§` 参照の本文転記・除去という追加作業が発生した。
- **横断規約（front matter）の適用対象を初回に洗い出せず last_modified が rules で漏れた**: design に `last_modified` + `adr`、rules に `adr` のみ、と最初に非対称化してしまい、追加レビューで rules にも `last_modified` を足す再修正になった。規約導入時に対象（design / rules）の体裁を最初から揃えるべきだった。
- **zsh で一括ファイル処理が word-split されず perl/sed が空振りした**: `for f in $FILES; do perl -i ...` が zsh では `$FILES` を分割せず全パスを 1 引数化して失敗。`git grep -l ... | while IFS= read -r f` へ切替で解消したが、ツール一括編集の初手で詰まった。
