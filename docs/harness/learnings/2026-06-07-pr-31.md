## ⚠️ Problem（詰まった点・制約）

- **マージが Phase 6 レビュー fix より先行（PR #29 と同型の再発）**: pokemon-data-reviewer が blocking（`mewtwo.yaml` の「M-B のみ解禁」コメントと修正後 `regulations: []` の矛盾）を検出し、follow-up commit `22c0902` で修正・push した。しかしオーケストレーターは最初の commit `9812a888` でマージ済みで、レビュー fix が **main に入らなかった**。implementation-workflow の「Phase 6 blocking → Phase 3 へ戻り修正してからマージ」が、worker / オーケストレーター分業のハンドオフでは保証されていない。
- **roster.moves のグローバルスコープ波及**: dragapult / hydreigon 用に追加した `shadow-ball` / `dark-pulse` / `nasty-plot` が、列挙外の metagross / rotom-wash / mewtwo の learnable にも生成上波及した（PokeAPI 上それらも習得するため）。決定論的生成の正しい挙動だが、技 1 つの追加が無関係種の moves を変える設計はレビューで都度説明が要る。
- **データ正当性をリポジトリ内で機械検証できない**: 解禁リストの真偽は出典コメント依拠で、tsc / カバレッジ / Biome のいずれも検証できない。memory に「暫定の可能性」既知があり、公開情報更新時の追従が人手頼みのまま。
- **自然言語コメントと生成データの整合が機械ゲート外**: `mewtwo.yaml` のコメントは tsc / check:individual で検証されず、データ観点レビュー（pokemon-data-reviewer）でしか拾えなかった。個体ファイルの説明文がデータ変更に追従漏れする経路が残る。
- **M-B 暫定リストの不確実性**: M-B は使用可能リスト未公開（2026-06-17 開始）のため allow を M-A 同集合で暫定設定。公開後に必ず更新が必要で、現状は「暫定」コメント頼みの技術的負債。
