# Phase 3 — ステータス調整の壁打ち

## 目的 / スコープ

育成済み個体の**ステータス調整**を機械的に支援する。実数値表示・耐久/火力指数・「確定耐え/確定数」からの能力ポイント逆算を提供し、構築のブラッシュアップを壁打ちできるようにする。アーキ詳細は [`architecture.md`](./architecture.md)。

## 前提（依存）

- Phase 0（`calc-stats`）/ Phase 1（種族データ）/ Phase 2（個体定義・検証）。
- 規約 `game-spec`（耐久/火力指数の定義を追記）。

## タスク

- [ ] `pokeform stat <path>`: 個体（YAML or 生成個体）の 6 実数値・性格補正後値・能力ポイント配分を表示（`--lang`）。
- [ ] 耐久/火力指数の算出（`src/domain/` に純関数）: 物理/特殊耐久指数・火力指数を `calc-stats` 上に実装。
- [ ] **ポイント逆算**: 「攻撃 ○○ の △△（技）を確定耐え」「素早さ □□ 抜き」等の条件 → 必要な能力ポイント配分を逆算（合計66/各32 制約内で解探索）。
- [ ] 調整提案レポート（現状 vs 目標、余剰ポイントの振り先候補）。
- [ ] 逆算結果が `calc-stats` と往復整合することを保証するテスト。

## この Phase で育てるハーネス

- rule: `game-spec` に耐久指数・火力指数・逆算の定義を追記。
- skill: ステータス調整提案スキル（壁打ち用）を canonical + symlink で作成。

## 受け入れ基準

- `pokeform stat` が実数値・指数を正しく表示。
- 逆算で求めた配分を `calc-stats` に通すと目標ラインを満たす（テストで検証）。
- `pnpm verify` 緑（逆算・指数の純関数は 100% カバレッジ）。

## 検証手順

1. 既知個体で実数値・耐久/火力指数の期待値一致を確認。
2. 「特定の技を確定耐え」条件で逆算 → 出力配分を `calc-stats` に通し、ダメージ計算上耐えることを確認。
3. 制約（合計66・各≤32）を超える要求には実現不能を報告する。

## リスク・備考

- ダメージ計算式（乱数・急所・タイプ一致等）の精度に依存。MVP では確定数判定に必要な範囲に絞り、将来の本格ダメージ計算は別計画（`02-<slug>`）へ。
- 逆算は探索問題。合計66 の制約下で一意でない解が出るため、提案は候補列挙 + 余剰振り先の指針を添える。
